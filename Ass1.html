Introduction
A markup language is a language that annotates text so that the computer can manipulate that text. Most markup languages are human readable because the annotations are written Win a way to distinguish them from the text itself. E.g SGML HTML, XML, and XHTML

List of Markup Languages

  1. Standard Generalized Markup Language (SGML
    2. )
    Standard Generalized Markup Language (SGML) was adopted by the International Standards Organization (ISO) in 1986. It quickly became the business standard for data storage and interchange. Advantages of SGML
    • Long-term viability as an ISO standard
    • Non-proprietary and platform-independent
    • Extremely flexible: supports user-defined tags reflecting the richness of documents
    Disadvantages of SGML
    • SGML tools are expensive, compared to those for HTML.
    • Creating DTDs with SGML is expensive - especial labor.
    • Writing SGML is extremely hard.
    • To put it bluntly, it is too elaborate for the ever-changing web.

    Back 2Top

  2. HyperText Markup Language (HTML)

    3. HTML is a simple data format used to create hypertext documents that are portable from one platform to another. HyperText Markup Language is the language of the web. All web pages are written in HTML. HTML defines the way that images, multimedia, and text are displayed in web browsers. It includes tags to connect documents (hypertext) and (forms) for making web documents interactive.
    Advantages of HTML
    • Easy to learn and code even for novice programmers
    • Free- you need not buy any software
    • Syntax is loose and more forgiving
    • Supported on almost every browser , if not all browsers
    • Widely used; established on almost every website, if not all websites
    Disadvantages of HTML
    • It cannot produce dynamic output alone, since it is a static language
    • You have to keep up with deprecated tags

    Back 2Top

  3. eXtensible Markup Language (XML)

    4. It is called “extensible” because it does not consist of a set of tags like HTML. It is really a “meta” language – a language from which to create other languages. It provides a consistent set of rules for creating these other languages and can be thought of as a lighter version of SGML. It represents an industry-wide effort to define which data are displayed (or printed), whereas HTML defines how a page is displayed.
    Advantages of XML
    • Provides metadata (data about information) that helps people find data
    • Data can be processed with inexpensive software
    • Simplifies the exchange of data between organizations
    • Provides information that aids automatic processing by electronic agents
    • It is less strict than SGML.
      • Disadvantages of XML
      • It does not consist of set of tags like HTML.
      • It is more strict than plain HTML.

    • eXtensible HTML (XHTML)

XHTML is the most current version of HTML. It combines aspects of HTML but, unlike HTML, is extensible like XML.
  • XHTML conforms to the stricter syntax rules of XML.
  • One beneficial aspect of XHTML, is that the more rigid structure identified by the DOCTYPE at the beginning of the document, facilitates the use of validators which can verify whether or not the document has correct XHTML syntax.
  • All XHTML tags and attributes must be in lowercase letters. HTML allows either case, and even mixed case.
  • All non-empty elements must have corresponding closing tags.
